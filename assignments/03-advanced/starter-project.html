Project Overview

This is your capstone assignment. You'll design and build a complete, professional design system from scratch. This isn't about copying—it's about strategy, architecture, and creating something usable by real developers.

💡 What makes this different: You'll learn not just CSS, but the thinking behind design systems. Why certain tokens exist, how systems scale, and how to make decisions that affect entire products.

What You'll Create

  • 50+ design tokens: Organized by category (colors, typography, spacing, etc.)
  • 10+ components: Reusable, well-documented UI pieces
  • Documentation: Professional README and design system guide
  • Demo site: Showcase of your system in action
  • Accessibility guide: WCAG compliance notes
  • Portfolio piece: Something to show employers

🎯 Challenge Tracks

Choose ONE of these three tracks. Each takes different amounts of time and focuses on different skills:

Track A: Brand New Design System

Recommended

Create a design system for a fictional company of your choice.

Focus: Understanding system architecture from the ground up
  • Define your company/brand (e.g., fintech startup, e-commerce, SaaS platform)

Track B: Multi-Framework System

Advanced

Same design system, implemented across multiple frameworks.

Focus: Framework-agnostic thinking

Track C: Enterprise System with Governance

Expert Level

Production-ready system with brand variations and governance guidelines.

Focus: Real-world complexity

  • Dark mode, light mode, and high-contrast themes

  • Governance documentation and contribution guidelines
  • Versioning and changelog
  • Integration tests for components

Step 2: Plan Your Tokens (1-2 hours)

Sketch out your token structure:

Phase 1: Project Setup

  • Create folder structure
  • Set up main files (index.html, design-tokens.css, components.css)
  • Initialize Git repository
  • Create README with project overview

Phase 2: Design Tokens

Define all tokens in design-tokens.css:

:root { /* Colors */ --color-primary: #0066cc; --color-secondary: #764ba2; /* Typography */ --font-size-h1: 32px; --font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto; /* Spacing */ --space-unit: 8px; --space-xs: var(--space-unit); /* Add more... */ }

Phase 3: Components

Build components using tokens in components.css:

.button { background-color: var(--color-primary); padding: var(--space-md) var(--space-lg); font-size: var(--font-size-body); /* ... */ } .button--secondary { background-color: var(--color-secondary); }

Phase 4: Demo Site

Create index.html showcasing all components and tokens

Phase 5: Documentation

Create DESIGN_SYSTEM.md with:

  • Token definitions and usage
  • Component specifications
  • Accessibility guidelines
  • Contribution guidelines
  • Code examples

✨ Success Tips

  • Start small: Begin with 5 basic tokens, expand from there
  • Build first, document second: Create working code first, then document what you built
  • Use your system: Build demo pages using only your tokens—no hardcoding
  • Test accessibility: Use axe DevTools or WebAIM checker
  • Get feedback: Show your work-in-progress to classmates or mentors
  • Version control: Commit regularly so you have a complete history
  • Document decisions: Write WHY you made choices, not just WHAT you did
